# Corporate Banking Credit Management System (CMS)

A full-stack web application designed to streamline the corporate credit request process. This system allows corporate clients to submit credit requests and enables bank administrators to review, approve, or reject them securely.

The application follows a modern **Microservices-ready** architecture with a decoupled Frontend and Backend, secured via **JWT (JSON Web Tokens)**.

## ğŸ“¥ Installation & Setup Guide

### 1. Database Setup
1.  Open your MySQL Workbench or Terminal.
2.  Create a database named `corp_banking_db` (or update `application.properties` to match your DB name).
3.  The tables will be auto-generated by Hibernate when the backend starts.

### 2. Backend Setup (Spring Boot)
1.  Navigate to the backend folder:
    ```bash
    cd backend/corp-banking-api
    ```
2.  Open `src/main/resources/application.properties` and update your MySQL credentials:
    ```properties
    spring.datasource.url=jdbc:mysql://localhost:3306/corp_banking_db
    spring.datasource.username=root
    spring.datasource.password=your_password
    ```
3.  Run the application:
    ```bash
    mvn spring-boot:run
    ```
    *The backend will start on `http://localhost:8080`.*

### 3. Frontend Setup (Angular)
1.  Navigate to the frontend folder:
    ```bash
    cd frontend/corp-banking-cms
    ```
2.  Install dependencies:
    ```bash
    npm install
    ```
3.  Start the development server:
    ```bash
    ng serve
    ```
4.  Open your browser and navigate to `http://localhost:4200`.
